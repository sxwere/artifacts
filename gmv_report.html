核心诊断

UV 流量下滑是本次 GMV 下降的绝对主因，贡献了 122.99% 的下跌量，且流量损失高度集中在 Paid Ads × Discount Campaign 的新客（流失 390 人，贡献渠道总下跌 39.39%）。CVR 的小幅改善（+0.20pp）部分对冲了压力，但体量不足以抵消 UV 的塌陷。

根因链

GMV −7.45%  →  UV 是主因（−123%）  →  新客流失（贡献 111%）  →  Paid Ads 渠道最大（39%）  →  Discount Campaign（−390 人，19%）  →  获客侧问题（需广告数据验证）

跨期对比：品牌 GMV 运作规律

与 8→9 月增长期对比：好月份 CVR 主驱动（+122%）盖过 UV 拖累（−33%），差月份 UV 拖累（−123%）远超 CVR 对冲（+36%）。这个品牌 GMV 走势的核心变量是流量获取能力。

01 问题定义
背景

7–12 月 GMV 走势如下：7 月最高（基线），8 月微跌，9 月回升（好月份），10 月持平，11–12 月连续下滑，12 月降幅最大（-7.45%）且绝对值最低。

分析目标：12 月 GMV 下滑，原因是流量问题、转化问题，还是客单价问题？同时选取 9 月作为"好月份"基准，用于跨期对比。7 月虽然 GMV 最高，但 7 月之前没有历史数据，无法构建用户分类的时间基准，故排除。

数据说明与处理

原始数据为用户行为日志，每行是一次访问记录，同一用户同月可能有多行（多渠道、多日期）。直接 count() 统计 UV 会把行为点击次数误算为用户人数，导致 UV 严重虚高。

修正方案：漏斗各步骤先 groupby('user_id').max() 取用户级别最大值（该用户这个月是否曾到达该步骤），再 sum() 得到去重人头数。UV 用 nunique()。

遇到的问题 · UV 算成了行为点击数

最初用 count() 统计 UV，实际上是在统计行数而非用户数，同一用户多次访问被重复计入。发现后改为 nunique()，同时对漏斗各步骤改用 groupby → max → sum 的去重逻辑，确保每个用户在每个步骤只计一次。

02 连锁替代法拆解三因子
方法说明

GMV = UV × CVR × AOV。三个因子在两个月之间同时变化，连锁替代法每次只替换一个因子，其余保持基期（11 月）值不变，通过前后差额剥离每个因子的纯贡献，交叉项被顺序分摊。

初步结论

UV 是压倒性的拖累主因，CVR 微升形成部分对冲但远不够，AOV 也在拖累。下一步需要分别验证 CVR 是否真的有问题，然后再聚焦 UV。

chain_substitution.py 
nov = monthly_analysis[monthly_analysis['month'] == '2025-11'].iloc[0]
dec = monthly_analysis[monthly_analysis['month'] == '2025-12'].iloc[0]

total_gmv_change = dec['gmv'] - nov['gmv']  # -204,351 元

# 每次只换一个因子，其余保持 11 月基期值
uv_effect  = (dec['uv']  - nov['uv'])  * nov['crv'] * nov['aov']
crv_effect = dec['uv'] * (dec['crv'] - nov['crv']) * nov['aov']
aov_effect = dec['uv'] *  dec['crv'] * (dec['aov'] - nov['aov'])

# 各因子贡献率（三者加总验证等于 100%）
uv_contribution  = uv_effect  / total_gmv_change  # 122.99%
crv_contribution = crv_effect / total_gmv_change  # -36.34%
aov_contribution = aov_effect / total_gmv_change  # 13.35%
03 CVR 排除链
为什么先排除 CVR

连锁替代显示 CVR 微升（+36% 正向贡献），但"微升"不代表 CVR 没有内部问题——它可能在某些客群或某些漏斗环节上存在结构性恶化，只是被整体数字掩盖了。在聚焦 UV 之前，需要先把 CVR 问题彻底排除干净。

排除链分三步，每步锁定一个可能的解释方向。

步骤一 · 用户结构切片

如果 CVR 升高是因为"高质量老用户占比提升"（结构性迁移），那么切开各客群看，老用户 CVR 应该明显高于新客，且 12 月老用户占比应该比 11 月更高。

排除结果

三个客群 CVR 差距极小（6.8%–7.8%），新客依然占 89%，没有发生结构性迁移。CVR 升高不是因为老用户占比提升。

遇到的问题 · 防数据穿越的用户分类

最初直接用全量数据判断用户"是否曾经来过"，隐患是如果函数被复用到其他月份，可能把未来数据混入历史判断。解决方案是函数内部先切片——只保留 target_month 及更早的记录再做判断，从根本上切断未来数据干扰。

user_tagging.py · 防数据穿越的用户分类 
def get_monthly_customer_tags(df, target_month, last_month):
    # 关键：先切片切断未来数据
    history_df = df[df["month"] <= target_month]
    target_users     = set(df[df["month"] == target_month]["user_id"])
    last_month_users  = set(df[df["month"] == last_month]["user_id"])
    before_last_users = set(df[df["month"]  < last_month]["user_id"])

    res = []
    for uid in target_users:
        if   uid not in last_month_users and uid not in before_last_users: tag = "新客户"
        elif uid in last_month_users:  tag = "留存客户"
        else:                           tag = "唤醒客户"
        res.append({"month": target_month, "user_id": uid, "customer_type": tag})
    return pd.DataFrame(res)

步骤三 · 折扣敏感度验证

Paid Ads × Discount Campaign 新客大量减少，一种可能的解释是：这批人进来了，但因为价格敏感（冲着折扣来的）所以不买，最终拖低了 CVR。用 9 月和 12 月新客的折扣使用率来检验这个假设。

两个月新客折扣使用率几乎相同，说明 12 月新客并不比 9 月更价格敏感。"来了不买"的解释被数据推翻。问题不在 CVR，而在于这批人根本没来——是获客侧的问题。

discount_validation.py · 折扣敏感度验证 
# 锁定 9 月和 12 月的新客子集
df_new_9  = pd.merge(df[df['month']=='2025-09'], tags_9[tags_9['customer_type']=='新客户'],  on=['user_id','month'], how='inner')
df_new_12 = pd.merge(df[df['month']=='2025-12'], tags_12[tags_12['customer_type']=='新客户'], on=['user_id','month'], how='inner')

# 去重到用户级别再算折扣使用率
for label, df_m in [('9月新客', df_new_9), ('12月新客', df_new_12)]:
    user_discount = df_m.groupby('user_id')['discount_applied'].max()

CVR 排除链结论

三步验证完成，CVR 问题彻底关闭：用户结构没有迁移、漏斗各环节稳定、新客不存在价格敏感性差异。 CVR 微升是正常波动，不是实质性变化。问题完全在 UV 侧。

04 UV 深挖：渠道 × 活动归因
分析思路

UV 下滑中，用户类型拆解显示新客流失是主要来源（贡献 111%，留存客户贡献 4%）。新客流失意味着获客侧出了问题，下一步锁定新客子集，下钻到渠道维度，再往下到 channel × campaign_type 二维交叉。

遇到的问题 · 渠道维度新客人数膨胀

渠道维度加总的新客人数（2,046）比 UV 层的新客流失人数（2,007）多。排查后发现：同一个新客用户当月可能从多个渠道进来，在渠道维度被计入多次。这个现象本身合理——用户可以多渠道触达，但渠道维度的数字只用于分析各渠道的相对贡献，不能用于验证新客总人数。

遇到的问题 · 下钻逻辑的口径一致性

渠道层和 campaign 层都锁定"新客户"子集分析。下钻时必须复用同一张 df_all_new（已 inner join 过用户标签），不能重新过滤主表——否则 campaign 层引入全量用户，数字会比渠道层膨胀，两层无法对齐。

渠道层结论

新客下降贡献从高到低：Paid Ads（39.4%）→ Organic（36.4%）→ Social（15.4%）→ Email（8.9%）。Paid Ads 是最大的单一渠道拖累。

Campaign 层结论

下钻到 channel × campaign_type 后，Paid Ads × Discount（−390 人，贡献 19%） 是单一最大下滑来源，其次是 Organic × Discount（−357 人，17.4%）。Discount 类活动在多个渠道都出现下滑，是跨渠道性的共性问题。

分析边界

获客量下降的深层原因（预算不足、广告素材效率下降、CTR/CPC 恶化、促销吸引力降低）需要广告投放数据才能验证，现有漏斗数据到此为止。

05 AOV 配角说明
在本项目中的角色

AOV 在 11→12 月贡献了 −13% 的拖累，量级远小于 UV（−123%），是配角。在跨期对比中，AOV 在好月份（8→9月）是 +11% 的助力，在差月份是 −13% 的阻力——方向随 GMV 走势同向变化，说明 AOV 可能是结果的一部分，而非独立原因。

AOV 的深层拆解（AUR × IPT）在独立的 AOV 专项分析中完成，结论是 IPT（购买件数）下降是主因，具体表现为中单向单件订单迁移。但由于该分析使用的数据表与本项目不同，结论仅作参考方向，不在本项目内直接引用。

06 跨期对比：8→9月增长期
为什么做这个对比

单看 12 月下滑只能说明"出了问题"，把好月份和差月份的驱动因子并排对比，才能看出这个品牌 GMV 的运作规律——到底是什么因子在决定涨跌。

运作规律

两段时期 UV 和 CVR 的方向一致（UV 拖累、CVR 正向），但力量对比不同决定了结果：好月份 CVR+AOV 合力（+133%）盖过 UV 拖累（−33%），GMV 净上升；差月份 UV 拖累（−123%）远超 CVR 对冲（+36%），且 AOV 转为阻力，GMV 净下滑。

这个品牌的 GMV 走势由 UV 和 CVR 的力量对比决定。流量获取能力的变化是核心变量。
